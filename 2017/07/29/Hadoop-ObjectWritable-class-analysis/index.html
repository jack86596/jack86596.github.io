Hadoop ObjectWritable 源码与相关单元测试介绍

Posted on 2017-07-29 | In Java , Hadoop

Hadoop IO模块是序列化和RPC的基础，该模块负责将java的各种对象转化为byte流以便在网络上传输，并且还要负责将byte流转回原对象。开篇我们先看一下hadoop IO模块类的关系图：




其他的很好理解，每种java的原生的类和容器都有一个对应实现了Writeable的类（IntWritable、MapFile等）。ObjectWritable 是hadoop.io模块里面一个比较特殊的类（用红色标出），它是一个通用的Writeable类，可以将任意的自定义类转为byte流，原因是因为会它会把将要转换的对象的类信息记录在自己的成员变量private Class declaredClass;中，然后序列化时，将类的名字，以string的形式一并写入byte流中。在收到对应的byte流后，解析程序可以知道类名string的长度，然后从byte流中获得类的信息，之后就是根据类的信息去以此解析这个类的所有成员变量的值。

ObjectWritable是Writeable的实现，必然要实现接口的方法： void write(DataOutput out) throws IOException;和void readFields(DataInput in) throws IOException;。ObjectWritable实现是通过调用类中的静态方法。

所以ObjectWritable最主要的逻辑集中在这2个静态函数上，先看writeObject方法。

接下来看看ArrayPrimitiveWritable.Internal的write方法，逻辑很简单，1）先将数组中存储的基本类型名称以string的方式写到byte流中（包括类名称string的长度）；2）将数组中元素的个数以Int的方式写进流中；3）然后根据基本类型来调用相应的写数组方法（writeBooleanArray、writeIntArray等）将数组写进流中。writeIntArray等就是简单的遍历各个元素，然后调用DataOutput的write基本类型的方法writeInt。

至此就已经完成了将一个对象写到byte流中，接着看看从byte流中解析获得对象的过程readObject。

解析过程到此就完成了。序列化和反序列化过程都是一一对应的。接下来就用Hadoop自带的测试用例来验证运行一下上述代码，看看写入byte流中的内容。测试类为TestArrayPrimitiveWritable。这个类主要测试了ArrayPrimitiveWritable的write和readFields方法。这2个方法基本覆盖了其他的Writeable和其衍生类的读写功能。

将这个182个字节传递给DataInput对象，然后分别用ObjectWritable.readObject和ArrayPrimitiveWritable处理，可以准确的将2个字符串的的2遍都读取出来，最后放进resultSet中，将resultSet和expectedResultSet对比，可以发现最终结果和期望的结果一致，表明解析数组成功了。

从这182字节的内容，我们可以看出ObjectWritable.readObject比ArrayPrimitiveWritable只是多了传递给ObjectWritable.readObject的declaredClass字符串的长度，其他都是一样的，调用ObjectWritable.readObject会隐式调用ArrayPrimitiveWritable的write方法。

这个测试用例只测试了ObjectWritable.readObject处理元素为基本类型的数组，并没有测试处理Writeable类型的对象。对于Writeable对象，其本身已经包含了序列化的函数write和readFields，并不需要借助ObjectWritable来实现序列化，但ObjectWritable可以处理任何类的序列化需求，当然也不排除处理Writeable及它衍生类。
